# Project Instructions

- Do not run builds unless the user explicitly asks for them.
- This includes `flutter build ...`, `docker compose build`, `docker compose up -d --build`, and `dart build ...`.
- Lightweight checks such as formatting or analyzer commands are allowed when needed for a code change.
- All app UI text, messages, comments, and code identifiers should be in English.
- Keep the native Flutter login and server-rendered HTML login aligned in fields, labels, layout, and visual styling.
- Keep `PROJECT_DOCUMENTATION.md` updated when project behavior, architecture, setup, or deployment details change.
- All web-related behavior and storage changes must remain multi-user safe.
