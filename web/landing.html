Zaptec charging history

Deeper charging insight than the provider interface offers.

Charge Ledger synchronizes your Zaptec chargers and charge sessions, then adds practical filters, totals, duration views, and cost calculations for reviewing charging behavior in more detail.

What Charge Ledger Does

Synchronizes Zaptec Data

After you sign in with Zaptec, the app can fetch your chargers and recent charge sessions from the Zaptec API for a richer local overview.

Filters Charge History

Filter sessions by charger, period, start time, or end time, then continue where you left off after a browser refresh.

Shows Totals and Costs

Review session count, energy, duration, and calculated cost based on your own kWh price and currency setting.

Customizes the Table

Reorder or hide charge session columns so the table matches how you inspect your charging data.

What Is Stored Server-Side

In web mode, Charge Ledger stores synchronized charger and charge history data in secure server-side storage so the web app can restore your view between sessions.

  • Server session metadata: customer id, email, and expiry time.
  • Zaptec charger metadata and synchronized charge sessions.
  • Your saved filters, kWh price, currency, and table column settings.
  • Zaptec passwords and access tokens are not stored in the server database.

Private by Design

Zaptec access tokens are kept in browser session storage for the active web session and sent to the server only when a synchronization request needs to call Zaptec. You can delete stored server-side data from the web app settings menu.
